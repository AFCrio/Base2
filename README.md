# Base2 — Система управління нарядами на чергування

Настільна програма для автоматизованого складання нарядів на чергування. Призначена для підрозділів, яким необхідно організовувати чергування особового складу з призначенням зброї, амуніції та транспорту.

## Можливості

- 📋 **Шаблони нарядів** — ієрархічна структура (Розділ → Група → Позиція → Часовий діапазон)
- 📝 **Наряди** — створення на основі шаблонів, незалежне версіонування
- 👤 **Довідники** — особи, зброя, транспорт, локації, звання, посади, пресети амуніції
- 🔒 **Знімки (snapshots)** — `RenderedLine` зберігає текст призначення на момент запису, зміни у довідниках не впливають на архівні наряди
- 🔍 **Фільтрація в реальному часі** у всіх довідниках
- ⚡ **Журнал змін шаблонів** — версіонування з описом правок
- ✅ **Валідація конфліктів** — перевірка подвійного призначення особи, зброї та транспорту

## Стек технологій

| Компонент       | Технологія                          |
|-----------------|-------------------------------------|
| Платформа       | .NET 10, Windows                    |
| UI              | Windows Forms (WinForms)            |
| База даних      | SQLite (`DutyOrder.db`)             |
| ORM             | Entity Framework Core 10            |
| DI              | `Microsoft.Extensions.DependencyInjection` |

## Структура проекту

```
Base2/
├── Models/          # Сутності EF Core (Person, Weapon, DutyOrder, …)
├── Data/            # AppDbContext, DatabaseInitializer
├── Services/        # OrderService, AppSettingsService
├── Helpers/         # NodeRenderer, RenderedLineBuilder
├── Forms/           # UI-форми (MainForm, TemplateEditorForm, …)
├── Migrations/      # EF Core міграції
├── AppServices.cs   # Точка реєстрації DI
├── Program.cs       # Точка входу
└── docs/            # Документація
```

### Основні форми

| Форма                  | Призначення                                               |
|------------------------|-----------------------------------------------------------|
| `MainForm`             | Головне вікно: список шаблонів та нарядів                |
| `TemplateEditorForm`   | Редактор ієрархічного шаблону (TreeView + властивості)   |
| `OrderCreatorForm`     | Створення/редагування наряду, призначення ОС             |
| `AssignmentForm`       | Діалог призначення особи, зброї, транспорту              |
| `ReferenceForm`        | Довідники (8 вкладок, CRUD)                              |
| `SettingsForm`         | Налаштування за замовчуванням                            |

## Швидкий старт

### Вимоги

- Windows 10/11
- .NET 10 Runtime або SDK

### Запуск

```bash
git clone https://github.com/AFCrio/Base2.git
cd Base2
dotnet run
```

База даних `DutyOrder.db` створюється автоматично поруч із виконуваним файлом. При першому запуску EF Core застосовує всі міграції.

### Перший сеанс роботи

1. Відкрити **«Довідники»** → заповнити Звання, Посади, Локації, Особи, Зброю, Транспорт
2. Натиснути **«Новий шаблон»** → побудувати ієрархію позицій
3. Натиснути **«Новий наряд»** → заповнити метадані та призначити ОС

Детальна покрокова інструкція: [docs/user-guide.md](docs/user-guide.md)

## Бізнес-логіка

```
Шаблон (структура позицій)
    └─► Наряд (конкретні дати + копія структури)
            └─► DutyAssignment (Особа + Зброя? + Транспорт? + RenderedLine)
```

- **Шаблон** описує структуру без дат. Зміни у шаблоні не впливають на вже створені наряди (версіонування).
- **Наряд** — копія структури шаблону з конкретними датами та часовими зонами.
- **Призначення** зберігає лише `PersonId`, `WeaponId`, `VehicleId` для аналітики, плюс `RenderedLine` — текстовий знімок для документу.

Детально: [docs/architecture.md](docs/architecture.md)
