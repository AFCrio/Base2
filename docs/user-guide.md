## 1. Перший запуск

Запустіть `Base2.exe` (або `dotnet run`). База даних `DutyOrder.db` створюється автоматично поруч із виконуваним файлом. Міграції застосовуються при кожному старті.

**Рекомендований порядок першого налаштування:**

```
Довідники → Звання → Посади → Локації → Особи → Зброя → Транспорт
               ↓
           Шаблон наряду
               ↓
           Наряди
```

## 2. Заповнення довідників

Натисніть **«Довідники»** на головному екрані. Відкриється вікно з вкладками.

### 2.1 Звання

1. Перейдіть на вкладку **«Звання»**
2. Натисніть **«Додати»**
3. Введіть назву звання (наприклад: *Рядовий*, *Сержант*, *Лейтенант*)
4. Вкажіть **Рівень** (число): використовується для сортування (чим вище — тим старше)
5. Натисніть **«Зберегти»**

> 💡 Рекомендовані рівні: Рядовий = 1, Молодший сержант = 2, … , Полковник = 20

### 2.2 Посади

1. Перейдіть на вкладку **«Посади»**
2. Натисніть **«Додати»** → введіть назву (наприклад: *Стрілець*, *Водій*, *Командир відділення*)
3. Натисніть **«Зберегти»**

### 2.3 Локації

Локації — це фізичні місця розміщення постів або зберігання зброї.

1. Вкладка **«Локації»** → **«Додати»**
2. Введіть **Назву** (наприклад: *КПП-1*) та **Адресу**
3. Натисніть **«Зберегти»**

### 2.4 Особи (особовий склад)

1. Вкладка **«Особи»** → **«Додати»**
2. Заповніть: Прізвище, Ім'я, По батькові
3. Оберіть **Звання** та **Посаду** зі списків
4. Поле **Ініціали** заповнюється автоматично
5. Натисніть **«Зберегти»**

> 💡 Налаштування за замовчуванням для нових осіб (звання та посада) можна задати у **«Налаштування»**.

### 2.5 Зброя

1. Вкладка **«Зброя»** → **«Додати»**
2. Введіть:
   - **Тип** (АК-47, ПМ тощо)
   - **Номер** (унікальний серійний номер)
3. Оберіть **Місце зберігання** (з довідника Локацій) — використовується для фільтрації при призначенні
4. Якщо зброя закріплена за особою — оберіть у полі **«Закріплено за»**
5. Натисніть **«Зберегти»**

### 2.6 Пресети амуніції

Пресети дозволяють автоматично підставляти тип і кількість патронів при призначенні зброї.

1. Вкладка **«Пресети зброї»** → **«Додати»**
2. Введіть **Тип зброї** (має збігатися з полем Тип у вкладці Зброя)
3. Введіть **Тип амуніції** (наприклад: *9мм*) та **Кількість** (наприклад: *16*)
4. Натисніть **«Зберегти»**

> 💡 Вбудовані пресети: ПМ → 9мм × 16, АК-47 → 5.45мм × 120

### 2.7 Транспорт

1. Вкладка **«Транспорт»** → **«Додати»**
2. Введіть: **Назву**, **Номер** (унікальний), **Тип** (наприклад: *Вантажний*, *Легковий*)
3. Натисніть **«Зберегти»**

## 3. Створення шаблону наряду

Шаблон — це структура наряду без конкретних дат. Один шаблон можна використовувати для багатьох нарядів.

1. На головному екрані натисніть **«Новий шаблон»**
2. Введіть назву та опис
3. Відкриється редактор шаблону з порожнім деревом

### 3.1 Структура дерева

Шаблон будується у вигляді ієрархії вузлів:

```
Шаблон "Варта понеділок"
├── [TimeRange] Перша зміна (00:00 – 06:00)
├── [TimeRange] Друга зміна (06:00 – 12:00)
├── [Section] КПП-1  (Локація: КПП-1)
│   ├── [DutyPosition] Начальник варти  (зброя: так, макс: 1)
│   └── [DutyPosition] Вартовий         (зброя: так, макс: 2)
└── [Section] Склад  (Локація: Склад №3)
    └── [DutyPosition] Вартовий         (зброя: так, макс: 1)
```

### 3.2 Типи вузлів

| Тип          | Де використовується                              |
|--------------|--------------------------------------------------|
| `TimeRange`  | Позначає зміну (перша/друга/третя зміна тощо)   |
| `Section`    | Пост або об'єкт; тут задається Локація          |
| `Group`      | Підгрупа позицій усередині розділу               |
| `DutyPosition` | Конкретна посада, сюди призначається ОС        |

### 3.3 Додавання вузлів

1. Клацніть **правою кнопкою** на дереві → **«Додати дочірній вузол»**
2. Оберіть тип вузла
3. У правій панелі заповніть властивості:

**Для Section:**
- Назва
- Локація (зброя фільтруватиметься за цією локацією)

**Для DutyPosition:**
- Назва посади у наряді
- ✅ Потребує зброї / ✅ Потребує амуніції / ✅ Потребує транспорту
- Макс. призначень (скільки осіб може займати цю позицію одночасно)

**Для TimeRange:**
- Мітка (наприклад: *Перша зміна*)

### 3.4 Зміна порядку вузлів

- Перетягніть вузол у потрібне місце
- Або: правий клік → **«Вгору»** / **«Вниз»**

### 3.5 Збереження шаблону

Натисніть **«Зберегти»**. Система зафіксує нову версію та запише рядок у журнал змін.

## 4. Створення наряду

1. На головному екрані оберіть шаблон у списку зліва
2. Натисніть **«Новий наряд»**
3. Заповніть метадані:

| Поле           | Опис                                             |
|----------------|--------------------------------------------------|
| Номер наряду   | Унікальний номер (наприклад: *№15/2026*)         |
| Дата наряду    | Дата документа                                   |
| Початок        | Дата та час початку чергування                   |
| Кінець         | Дата та час завершення чергування                |
| Командир       | Прізвище та посада командира наряду              |

4. Натисніть **ОК** — відкриється форма наряду зі структурою, скопійованою з шаблону

### 4.1 Налаштування часових зон (змін)

Якщо шаблон містить вузли `TimeRange`, для кожної зміни потрібно вказати конкретний час:
- Оберіть зміну у дереві
- Вкажіть час початку та кінця за допомогою `DateTimePicker`

## 5. Призначення особового складу

### 5.1 Відкрити форму призначення

1. Клацніть на позицію типу `DutyPosition` у дереві наряду
2. Натисніть **«Призначити»**

### 5.2 Пошук особи

- Введіть прізвище (або частину) у поле пошуку — список відфільтрується в реальному часі
- Оберіть особу зі списку

### 5.3 Вибір зброї (якщо потрібно)

- Список містить лише зброю, що зберігається у Локації батьківського Розділу
- Список показує зброю, закріплену за обраною особою (спочатку), потім іншу доступну
- Оберіть одиницю зброї

### 5.4 Амуніція (якщо потрібно)

- Автоматично підставляється з пресету для обраного типу зброї
- Можна змінити тип та кількість вручну

### 5.5 Транспорт (якщо потрібно)

- Оберіть ТЗ зі списку доступних

### 5.6 Підтвердження

Натисніть **ОК**. Система:
1. Перевірить, чи не призначена ця особа вже на іншу позицію в цьому наряді
2. Перевірить, чи не зайнята обрана зброя/транспорт
3. Сформує `RenderedLine` — текстовий знімок (наприклад: *Іванов І.І., рядовий, ПМ №АА1234, 16 патр. 9мм*)
4. Збереже призначення

> ⚠️ **Увага:** Якщо особа або ресурс вже зайняті, система покаже помилку і не дозволить зберегти.

## 6. Типові сценарії

### Змінити особу у призначенні

1. Відкрийте наряд
2. Клацніть на позицію з призначенням
3. Натисніть **«Змінити призначення»** (або видаліть поточне та додайте нове)

### Повторно використати структуру наряду

Оберіть той самий шаблон → **«Новий наряд»** → новий наряд матиме ту саму структуру з чистими призначеннями.

### Редагувати шаблон, не зачіпаючи старі наряди

Відкрийте шаблон → **«Редагувати»**. Зміни зафіксуються як нова версія. Існуючі наряди містять знімок старої версії структури і не зміняться.

### Деактивувати шаблон

Якщо шаблон більше не потрібен, але наряди на його основі існують — виберіть шаблон → **«Деактивувати»**. Шаблон зникне з активного списку, але наряди збережуться.

### Видалити шаблон

Видалення доступне лише якщо на основі шаблону **не створено жодного наряду**.

### Знайти особу у довіднику

Відкрийте **«Довідники»** → вкладка **«Особи»** → введіть прізвище у поле фільтра.

### Перевірити, яка зброя є в наявності

**«Довідники»** → вкладка **«Зброя»** → фільтрація за назвою або локацією.

## 8. Часті питання (FAQ)

**Зброя не з'являється у списку при призначенні**  
→ Зброя фільтрується за локацією батьківського Розділу (`Section`). Переконайтесь, що у вузлі `Section` обрана правильна Локація **і** що зброя має цю саму Локацію у довіднику.

**Не можу видалити особу з довідника**  
→ Особа бере участь у наряді. FK `RESTRICT` блокує видалення. Спочатку видаліть або замініть усі її призначення.

**Не можу видалити шаблон**  
→ На основі цього шаблону існують наряди. Видалення заблоковано (FK `RESTRICT`). Деактивуйте шаблон.

**Де зберігається база даних?**  
→ Файл `DutyOrder.db` знаходиться у тій самій папці, що й `Base2.exe`.

**Як зробити резервну копію?**  
→ Скопіюйте файл `DutyOrder.db` у безпечне місце (можна просто на флешку).

**Чи можна відкрити базу даних іншою програмою?**  
→ Так, будь-яким SQLite-клієнтом (наприклад, [DB Browser for SQLite](https://sqlitebrowser.org/)).

**Зміна прізвища особи у довіднику впливає на старі наряди?**  
→ Ні. Старі наряди містять `RenderedLine` — текстовий знімок, зроблений у момент призначення. Нові призначення відображатимуть актуальне прізвище.
