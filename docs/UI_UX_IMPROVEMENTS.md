# Пропозиції покращення UI/UX

## 🔴 Критичні проблеми

### 1. Втрата даних без попередження
**Проблема:** Зміни зберігаються відразу в БД (видалення призначення, редагування вузла). Немає можливості скасувати дію.
**Рішення:**
- Додати діалог «Зберегти зміни?» при закритті форм з незбереженими змінами
- Реалізувати патерн «Зберегти / Скасувати» замість миттєвого збереження
- Додати кнопку «Скасувати останню дію» (Undo) для критичних операцій

### 2. Зависання UI при завантаженні даних
**Проблема:** Усі операції синхронні. UI блокується при завантаженні великих списків (1000+ осіб, зброї).
**Рішення:**
- Використати `async/await` для завантаження даних
- Додати індикатор завантаження (ProgressBar або спінер)
- Завантажувати дані у фоновому потоці

### 3. Глибоке вкладення модальних діалогів
**Проблема:** Ланцюжок: MainForm → OrderCreatorForm → AssignmentForm → PersonEditDialog (3+ рівні вкладеності). Користувач губиться.
**Рішення:**
- Зменшити глибину вкладення до 2 рівнів максимум
- Інлайн-редагування даних (DataGridView EditMode) замість окремих діалогів для простих сутностей
- Використовувати панелі замість діалогів де можливо

## 🟠 Важливі покращення

### 4. Відсутність пошуку та фільтрації у головній формі
**Проблема:** Немає пошуку наказів за номером, датою, командиром. У великому списку важко знайти потрібний наказ.
**Рішення:**
- Додати поле пошуку над таблицею наказів
- Фільтр за діапазоном дат
- Сортування за стовпцями (клік на заголовок)

### 5. Відсутність гарячих клавіш
**Проблема:** Робота виключно мишкою. Досвідчені користувачі не мають швидкого доступу до частих операцій.
**Рішення:**
- `Ctrl+S` — Зберегти
- `Ctrl+N` — Новий наказ/шаблон
- `Ctrl+F` — Пошук
- `Delete` — Видалити вибраний елемент
- `F5` — Оновити дані
- `Ctrl+Z` — Скасувати (де реалізовано)

### 6. Відсутність візуального зворотного зв'язку
**Проблема:** Після збереження, створення, видалення — немає чіткого повідомлення про успіх (лише закриття діалогу).
**Рішення:**
- Додати StatusBar у нижній частині форм з повідомленнями (✅ Збережено, ⚠️ Увага тощо)
- Анімація/підсвічування змінених записів у таблиці
- Тост-повідомлення для неблокуючих сповіщень

### 7. Покращення форми призначення (AssignmentForm)
**Проблема:** Таблиця показує лише непризначених осіб. Немає інформації хто вже призначений і куди.
**Рішення:**
- Додати вкладку «Усі призначення» з можливістю перепризначення
- Показувати «зайняті» особи сірим кольором з позначкою куди призначені
- Додати попередній перегляд відрендереного рядка перед збереженням

### 8. Покращення редактора шаблонів (TemplateEditorForm)
**Проблема:** Немає попереднього перегляду повного наказу. Система плейсхолдерів потребує запам'ятовування синтаксису.
**Рішення:**
- Додати панель «Попередній перегляд» з рендерингом шаблону (з тестовими даними)
- Автодоповнення плейсхолдерів при введенні `{` (випадаючий список)
- Підсвічування невалідних плейсхолдерів червоним
- Кнопка «Вставити плейсхолдер» з випадаючим меню

## 🟡 Бажані покращення

### 9. Покращення ReferenceForm (Довідники)
**Проблема:** 1980 рядків коду, 7 вкладок. При перемиканні — перезавантаження всіх даних. Немає пагінації.
**Рішення:**
- Lazy-завантаження даних при перемиканні вкладки (а не всіх одразу)
- Пагінація для великих списків (50/100 записів на сторінку)
- Вбудоване редагування в DataGridView для простих полів (замість діалогу)
- Кнопки Імпорт/Експорт (CSV/Excel) для масового оновлення довідників

### 10. Покращення TreeView (дерево наряду)
**Проблема:** Повне перебудування дерева при кожній зміні. Великі шаблони (50+ вузлів) повільно перемальовуються.
**Рішення:**
- Оновлювати лише змінений вузол, а не всю структуру
- Додати згортання/розгортання всіх вузлів одночасно
- Пошук по дереву з підсвічуванням знайдених вузлів
- Іконки вузлів у вигляді зображень замість emoji

### 11. Друк та експорт
**Проблема:** Немає можливості роздрукувати наказ або експортувати в Word/PDF.
**Рішення:**
- Попередній перегляд друку (Print Preview)
- Експорт у Word (.docx) через бібліотеку
- Експорт у PDF
- Копіювання тексту наказу в буфер обміну

### 12. Адаптивність та доступність
**Проблема:** Фіксовані розміри деяких форм. Немає підтримки DPI-масштабування. Відсутні підказки.
**Рішення:**
- Використовувати DPI-aware макети (AutoScale)
- Додати ToolTip підказки до всіх кнопок
- Збільшити контрастність кольорів для вузлів дерева
- Підтримка високого контрасту (системна тема Windows)

### 13. Покращення форми налаштувань
**Проблема:** Мінімальний функціонал (лише звання/посада за замовчуванням).
**Рішення:**
- Налаштування шляху до бази даних
- Резервне копіювання/відновлення БД
- Налаштування формату дат
- Журнал змін шаблонів (TemplateChangeLog існує, але не відображається)
- Про програму (версія, ліцензія)
